ESP32 Home 仓库结构说明

当前仓库围绕“服务端主控 + 局域网客户端面板 + 文档中心”来组织。服务端负责采集、网页、自动化和串口红外桥接；客户端负责按钮、旋钮、RGB 指示灯和 HTTP 控制；共享文档只保留已定方案，未定的 LVGL 画面和 MQTT 细节先挂起。

子项目分工

服务端

esp32_home_server/ 负责传感器采样、自动化、网页 API、风扇/窗帘执行和串口红外桥接。

客户端

esp32_home_client/ 负责按键、旋钮、RGB、LCD1602，以及通过 HTTP 控制服务端。

文档

doc/ 是统一文档中心。协议、需求和结构图都优先写在这里，不再在子项目里散落副本。

暂缓项

LVGL 8pin 显示界面尚未定稿，因此先屏蔽相关显示实现；MQTT 也仅保留占位接口，不作为当前联调主链路。

项目结构关系图

下图按“仓库根目录 -> 核心子项目 -> 运行关系”展示当前工程结构，便于新成员快速定位开发入口与联调链路。

esp32_home/（仓库根）
统一管理服务端固件、客户端固件、红外实验工程与文档。
▼ 子项目拆分
esp32_home_server/
主控服务端：传感器采样、自动化引擎、本地 Web API、串口红外桥接。
esp32_home_client/
控制面板端：按键/旋钮输入、RGB/LCD 输出、通过 HTTP 调服务端控制接口。
doc/
项目文档中心：需求、流程、架构、协议说明统一沉淀。
▼ 运行时交互
客户端 → 服务端：HTTP /api/status 拉取状态，/api/control 下发控制。
服务端 → ESP8266 红外板：串口发送文本命令，ESP8266 负责协议解释与红外发射。
服务端 ↔ 浏览器：通过 web/index.html 提供本地可视化与调试入口。

当前约定

服务端联网 Wi-Fi 已固定为 iPadmini / lbl450981。离线时仍回退到热点 esp32-server / lbl450981。
客户端主链路只走 HTTP，不依赖 MQTT。MQTT 当前只保留占位代码与说明，后续单独设计再恢复。
红外控制简化为字符串命令。服务端只负责向 ESP8266 发字符串，由 ESP8266 自己解释协议与动作。
8pin TFT 以后会改成 LVGL 界面，但目前界面尚未设计，因此显示逻辑先屏蔽，不让未定 UI 反向污染协议和固件结构。
