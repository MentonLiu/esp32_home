# ESP32 智能家居项目 - 2026年3月24日变更记录

## 概述
本次更新修复了三个关键问题：电机与舵机冲突、网页电机控制面板移除、以及添加了完整的USB串口诊断日志系统。

---

## 🔧 修復清單

### 1. **修复电机与舵机 PWM 冲突** ⚡
**问题症状**：
- 控制直流电机时舵机会同时运动
- 程序概率性死机

**根本原因**：
- 旧配置：电机使用 LEDC 通道 4
- ESP32Servo 库会自动分配 LEDC 通道（通常是 0-3）
- 通道争用导致 PWM 信号干扰
 
**修复方案**：
- ✅ 将电机 LEDC 通道从 **4** 改为 **5**（`CentralProcessor.cpp`）
- 舵机保持自动分配，现在不会与电机冲突
- 蜂鸣器使用通道 6（保持不变）
- **LEDC 通道分配**: 0-3 (舵机自动) | 5 (电机) | 6 (蜂鸣器)

**文件修改**：
- `src/CentralProcessor.cpp`: 行 14 → `fan_(pins::FAN_PWM, 5)`

### 3. **新增 USB 串口日志系统** 📝
**功能特性**：
- ✅ 完整的日志类 `Logger`，支持 4 个日志级别 (DEBUG/INFO/WARN/ERROR)
- ✅ 时间戳、标签、格式化输出
- ✅ 便捷的宏函数：`LOG_DEBUG()`, `LOG_INFO()`, `LOG_WARN()`, `LOG_ERROR()`
- ✅ 串口波特率 115200，与主循环共享同一串口

**日志输出格式**：
```
[123.456] [INFO] [INIT] 初始化传感器...
[124.789] [DEBUG] [DHT] DHT 传感器初始化完成 (Pin=13)
[130.145] [WARN] [DHT] 读取失败 - 请检查: 1) DHT11接线 2) 电源和GND是否正确...
```

**新增文件**：
- `include/Logger.h` - 日志系统头文件
- `src/Logger.cpp` - 日志系统实现

**集成位置**：
- `src/CentralProcessor.cpp` - 系统启动日志，LEDC 通道分配确认
- `src/Sensor.cpp` - 传感器初始化、DHT 读取诊断、定期状态输出

### 4. **DHT11 诊断信息** 🔍
**诊断内容**：
- ✅ DHT11 初始化日志
- ✅ **读取失败时输出详细提示**：检查接线、电源/GND连接、上拉电阻(4.7kΩ)
- ✅ 每 30 秒输出一次完整的传感器状态快照
- ✅ 开启日志等级后，用户可通过串口监看 DEBUG 信息

## 📋 测试建议

### 1. 电机/舵机冲突修复验证
```
测试步骤：
开启网页控制
1. 只控制舵机（打开/关闭窗帘）→ 舵机单独动作，无闲置
2. 通过 API 控制电机（可用 curl 或自定义脚本）→ 电机独立运行，舵机不干扰
3. 同时控制电机和舵机 → 两者独立运行，无冲突
预期结果：程序稳定运行 > 5 分钟，无死机
```

### 2. DHT11 数据丢失排查
连接 USB 串口，打开串口监视器（115200 波特率）
```
观察日志输出：
- 看是否出现 "[WARN] [DHT] 读取失败" 消息
- 查看每30秒输出的传感器状态，确认温度值更新
- 如果一直显示 T=0.0°C，检查：
  ① DHT11 数据线(GPIO13) 是否插正确
  ② 电源和 GND 是否有公共接地（单独供电时很重要！）
  ③ 数据线是否有 4.7kΩ 上拉电阻
```

### 3. 电机 API 控制测试
```bash
# 开启电机（90%速度）
curl -X POST http://192.168.x.x/api/control \
  -H "Content-Type: application/json" \
  -d '{"device":"fan","speedPercent":90}'

# 关闭电机
curl -X POST http://192.168.x.x/api/control \
  -H "Content-Type: application/json" \
  -d '{"device":"fan","mode":"off"}'
```

## 📌 重要硬件提醒（针对 DHT11 问题）

用户提到"为外设和主控提供了单独供电"后 DHT11 不显示数据，**最常见的原因**：

1. **🔴 地线未共接** ← **最可能的原因**
   - 独立供电时，DHT11、ESP32、电机驱动的 GND 必须连在一起
   - 没有公共地线会导致数据线通讯失败

2. **📍 接线检查清单**
   - DHT11 VCC → 3.3V 电源
   - DHT11 GND → ESP32 GND（**非常重要！**）
   - DHT11 DATA → GPIO 13（带 4.7kΩ 上拉电阻到 3.3V）
   - 电机驱动 GND → 也要连到 ESP32 GND

3. **✅ 验证步骤**
   - 使用万用表测量 DHT11 GND 和 ESP32 GND 之间的电阻 → 应该 < 1Ω
   - 启动后观察串口日志，是否有 DHT 初始化和读取信息
