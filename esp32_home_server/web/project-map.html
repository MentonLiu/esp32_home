ESP32 Home 项目结构关系图

这个版本完全按 `/doc` 重建，核心链路是“传感器采集 -> 标准化数据 -> 本地网页 / MQTT 上行 -> 控制命令处理 -> 执行器动作”，再叠加自动化规则和网络模式切换。

模块分层

硬件层

Sensor.cpp 负责 DHT、光照、MQ2、火焰采样，Controllerr.cpp 负责风扇 PWM、双舵机窗帘、蜂鸣器和红外桥接串口。

标准化层

SensorDataProcessor 把原始采样转成统一状态 JSON；ControllerCommandProcessor 把网页、MQTT、自动化指令转成统一设备动作。

连接层

ConnectivityManager 管理路由器 Wi-Fi、esp32-server 热点、本地 WebServer 和云端 MQTT，按 30 秒周期重评估网络模式。

编排层

AutomationEngine 负责时间窗帘、烟雾报警、火焰报警；CentralProcessor 统一初始化并驱动整个系统主循环。

运行流程

启动时先尝试连接 home-WiFi；如果联网成功，进入 cloud 模式，Web 页面和 MQTT 同时可用。
如果联网失败，系统自动启动热点 esp32-server，密码 lbl450981，并继续提供同一套 HTTP 页面和接口。
主循环每 0.5 秒采样一次传感器，并在云端模式下将标准化数据发布到 esp32/home/sensors。
控制命令可来自网页 /api/control、云端 MQTT 控制主题或自动化规则，都会走统一的命令处理器。
自动化部分会在每天 07:00 打开窗帘、22:00 关闭窗帘；高烟雾时强制风扇最高速，火焰持续时触发蜂鸣器和云端火警上报。
