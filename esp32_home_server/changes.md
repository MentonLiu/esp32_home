# 服务端变更记录

日期：2026-03-23

## 今日改动（自动化与可维护性）

1. 修复窗帘舵机潜在通道冲突：风扇 PWM 通道改为独立 LEDC 通道（避免与舵机资源竞争）。
2. 修复火焰状态回落不稳定：火焰检测由单阈值改为滞回阈值（触发/释放双阈值）。
3. 修复页面火焰显示兼容性：前端对 `flameDetected` 增加布尔/数字/字符串兼容解析。
4. 新增温度联动风扇规则：高温触发风扇高档，带滞回与冷却保护。
5. 新增光照联动窗帘规则：白天强光防眩、弱光增亮，带滞回、冷却与手动覆盖窗口。
6. 新增手动窗帘覆盖保护：记录最近一次非自动化窗帘操作，30 分钟内自动化不接管。
7. 新增红外气候联动规则：高温或夜间高湿持续触发 `ac_mode` / `ac_temp_down`，带持续判定与冷却保护。
8. 为自动化新增 IR 直发接口：控制处理器提供 `sendIrCommand` 供规则层直接调用。
9. 对核心模块补充大量中文注释：覆盖自动化、控制器、联网、传感器、页面脚本与样式区块。
10. 完成多轮构建验证：`pio run` 持续通过。
11. 风扇控制明确切换为“直流电机 PWM 调速”方案，并调整为更通用的 2kHz PWM 参数。
12. 补充 ULN2003A 对接说明：`GPIO18 -> IN1`、`OUT1 -> 电机负极`、`COM -> +5V`、系统共地。
13. 明确引脚对接约束：风扇 PWM 不与红外串口 RX（GPIO15）复用。
14. 删除风扇独立电源开关控制：移除 `fanPowerOn` 状态与 `power:on/off` 命令，统一改为 `mode/speedPercent` 控制。

日期：2026-03-21

## 本轮调整

1. 根据最新项目要求，将服务端 Wi-Fi 主网络固定为 `iPadmini / lbl450981`。
2. 保留 `esp32-server / lbl450981` 作为兜底热点。
3. 将红外控制协议收敛为“通过串口向 ESP8266 发送字符串命令”。
4. 本地控制接口统一到 `GET /api/status` 与 `POST /api/control`。
5. 状态接口新增 `fanPowerOn`，明确风扇电源与风速是两个状态。
6. 风扇只有在通电状态下才允许调速，自动化逻辑也遵守这一规则。
7. 删除服务端网页与项目结构页的关联，结构说明改为仓库根目录的 `project-map.html`。
8. MQTT 代码暂时保留占位能力，但当前主流程不启用。
9. 为客户端联调同步清理了时间展示依赖字段，避免双端协议出现无意义负担。
