# ESP32 Home Client 需求说明

本文基于当前客户端代码实现整理，作为联调与装配说明。

## 1. 当前定位

- 客户端是控制面板，不承担传感器采样与自动化主控。
- 客户端通过 HTTP 与服务端通信：
  - 轮询状态：`GET /api/status`
  - 下发控制：`POST /api/control`
- 客户端当前输入方式为 4 个独立按键，不包含旋钮控制逻辑。

## 2. 设备与接线

### 2.1 输入设备

| 设备   | 引脚   | 功能                              |
| ------ | ------ | --------------------------------- |
| 按钮 1 | GPIO32 | 窗帘状态循环：0/45/90/135/180     |
| 按钮 2 | GPIO33 | 风扇档位循环：off/low/medium/high |
| 按钮 3 | GPIO26 | 预留                              |
| 按钮 4 | GPIO13 | 预留                              |

说明：按键使用 `INPUT_PULLUP`，接法为 GPIO 与 GND。

### 2.2 输出设备

| 设备             | 引脚                        | 功能               |
| ---------------- | --------------------------- | ------------------ |
| RGB-R            | GPIO25                      | 烟雾等级指示       |
| RGB-G            | GPIO14                      | 烟雾等级指示       |
| RGB-B            | GPIO15                      | 烟雾等级指示       |
| LCD1602 SDA/SCL  | GPIO21/GPIO22               | 展示联网与传感摘要 |
| TFT SCLK/MOSI    | GPIO18/GPIO23               | TFT SPI 数据总线   |
| TFT RST/DC/CS/BL | GPIO17/GPIO16/GPIO27/GPIO19 | TFT 控制与背光     |

## 3. 功能说明

### 3.1 状态拉取与显示

- 轮询周期约 800ms。
- 从服务端状态中读取温湿度、光照、MQ2、烟雾等级、风扇状态、窗帘角度。
- LCD1602 与 TFT 按刷新周期显示当前状态。
- RGB 根据 `smokeLevel` 变化显示颜色等级。

### 3.2 控制行为

- 按钮 1：发送窗帘角度命令
  - `{"device":"curtain","angle":0|45|90|135|180}`
- 按钮 2：发送风扇模式命令
  - `{"device":"fan","mode":"off|low|medium|high"}`
- 按钮 3/4：预留，不发送设备控制。

### 3.3 网络与发现

- 客户端优先连接家庭 Wi-Fi：`HW-2103 / 20220715`
- 失败后回退连接服务端热点：`esp32-server / lbl450981`
- 在热点模式下，优先访问 `http://192.168.4.1`
- 在家庭 Wi-Fi 下，优先通过 mDNS 查询 `esp32-home-server.local`

## 4. 与服务端协议约束

- 风扇控制仅支持 `mode` 或 `speedPercent`，不使用 `power` 字段。
- 状态同步以 `fanMode` 与 `fanSpeedPercent` 为准。
- 控制失败时，按服务端返回的 `ok/type/message` 处理。

## 6. 硬件细节

- 主控：ESP32
- 输入：4 个按键（INPUT_PULLUP）
- 指示：RGB 三色灯
- 显示：2.0 TFT 8pin + LCD1602 I2C
