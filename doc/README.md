# 项目文档中心

`doc/` 是整个仓库的统一文档入口。和固件实现有关的需求、协议、结构图都优先维护在这里，避免服务端和客户端各写一份相互漂移。

## 文档分区

- `server/`
  服务端需求、流程和架构图。
- `client/`
  客户端需求与架构图。
- `shared/`
  跨子项目共享的协议、联调约定和对接说明。

1. [`../project-map.html`](../project-map.html)

## 当前文档边界

- Wi-Fi 已固定为 `iPadmini / lbl450981`
- 客户端主链路是 HTTP，不把 MQTT 当作必须项
- MQTT 仍可在文档中保留“后续规划”说明，但不写成当前既成实现
- 红外链路统一描述为“服务端串口发送字符串命令到 ESP8266”
- LVGL 8pin TFT 相关内容仅保留“后续待实现”说明，不写未定界面细节

## 维护约定

1. 协议和联调说明优先写在 `shared/`。
2. 需求原文保留在 `server/` 和 `client/`，尽量不在子项目里再复制一份。
3. 当实现与旧文档冲突时，先更新文档再改代码。
