# 项目文档中心

`doc/` 是整个仓库的统一文档入口。它的目标不是重复每个子项目 README，而是回答这些更高层的问题：

- 这个仓库现在到底以什么链路在运行
- 哪些模块已经实现，哪些还在规划
- 先看什么文档最容易快速理解项目
- 接线、联调和协议应该以哪份说明为准

## 文档分区

- `快速上手.md`
  面向新接手者的最短阅读路径和启动顺序。
- `版本演进与里程碑.md`
   基于 git 提交历史整理的演进阶段说明。
- `server/`
  服务端需求、流程和架构图。
- `client/`
  客户端需求和架构图。
- `shared/`
  跨子项目共享的系统总览、协议、接线和联调说明。

## 哪些文档最权威

当前建议按这个优先级理解文档：

1. 根目录和 `doc/shared/` 的现状说明
2. `esp32_home_server/README.md`
3. 历史需求文档
4. 子项目里可能滞后的旧说明

原因是项目最近做过几次协议和控制口径收敛，部分旧设计还留在历史文档里，但并不等于当前代码实现。

## 当前文档边界

- Wi-Fi 已固定为 `iPadmini / lbl450981`
- 服务端离线热点为 `esp32-server / lbl450981`
- 当前主链路是 HTTP，不把 MQTT 当成必需路径
- 红外链路统一描述为“服务端串口发送字符串命令到 ESP8266”
- `esp8266_ir_control/` 当前仍是占位工程，文档应明确标出
- LVGL 8pin TFT 保留为后续方向，不写成既成实现

## 维护约定

1. 协议和跨项目联调说明优先写在 `shared/`
2. 新成员阅读入口优先更新 `快速上手.md`
3. 当实现与旧文档冲突时，以“当前可运行实现”优先，并尽快回写文档
4. 不把未完成模块写成已完成能力
