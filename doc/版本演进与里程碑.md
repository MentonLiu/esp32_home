## 分阶段说明

### 阶段 1：项目初始化与需求沉淀（2026-03-15）

- 建立项目基础目录。
- 完成功能流程图和功能要求文档。
- 形成“先文档后实现”的初始节奏。

### 阶段 2：服务端首版与红外桥接方向明确（2026-03-16 ~ 2026-03-17）

- 落地 ESP32 家庭服务端首版。
- 红外控制改为“服务端串口发字符串命令，ESP8266 负责执行”。
- 移除旧兼容路径，开始统一红外链路语义。

### 阶段 3：架构重构与平台迁移（2026-03-17 ~ 2026-03-19）

- 迁移 ArduinoJson 到 v7 语法。
- 拆分服务逻辑，调整引脚、WebFS、时钟与模块边界。
- 工程迁移到 ESP32-S3。
- Web 控制台与结构页解耦，改由 LittleFS 承载本地控制页面。

### 阶段 4：文档体系建设与结构整理（2026-03-20 ~ 2026-03-22）

- 按文档重建项目结构并刷新文档。
- 补全中文注释，新增项目结构说明页。
- 新建 `esp32_ir_Study` 与 `esp8266_ir_control` 配套目录。
- 增加共享接线说明，开始形成统一联调文档入口。

### 阶段 5：可用性优化与控制模型收敛（2026-03-23）

- 增强自动化规则并补全中文文档。
- 修正风扇 PWM 调速实现细节。
- 将风扇控制统一为模式与速度语义。
- 优化本地控制响应，降低执行器阻塞和联动风险。

## 当前阶段结论

项目当前主线不是“继续堆功能”，而是“确保服务端本地链路稳定可用”：

- 主链路：HTTP + 本地网页 + 串口红外桥。
- 主工程：`esp32_home_server/`。
- 客户端定位：第二控制入口。
- 红外执行端：`esp8266_ir_control/` 仍待实现。
