# 快速上手

这份文档的目标不是覆盖全部细节，而是帮助你在最短时间内把仓库跑起来，并知道先看哪里。

## 先理解什么

当前仓库的中心是 [`esp32_home_server/`](../esp32_home_server/README.md)。

如果你时间有限，只需要先理解这三件事：

1. 服务端提供网页和 HTTP API。
2. 浏览器和 `esp32_home_client` 都是服务端的控制入口。
3. 红外命令目前只是被服务端转发到 ESP8266，真正的红外发射端还没做完。

## 推荐启动顺序

1. 阅读 [`../readme.md`](../readme.md)
   建立仓库整体概念。
2. 阅读 [`shared/系统总览与联调指南.md`](shared/%E7%B3%BB%E7%BB%9F%E6%80%BB%E8%A7%88%E4%B8%8E%E8%81%94%E8%B0%83%E6%8C%87%E5%8D%97.md)
   理解运行链路和设备关系。
3. 进入 [`../esp32_home_server/README.md`](../esp32_home_server/README.md)
   看服务端模块、接口和当前能力。
4. 检查 [`shared/项目接线说明.md`](shared/%E9%A1%B9%E7%9B%AE%E6%8E%A5%E7%BA%BF%E8%AF%B4%E6%98%8E.md)
   确认接线和供电设计。

## 先跑服务端

进入 `esp32_home_server/` 后，推荐顺序：

1. `pio run`
2. `pio run -t buildfs`
3. `pio run -t upload`
4. `pio run -t uploadfs`
5. `pio device monitor -b 115200`

验证项：

- 能连接家庭 Wi-Fi 时，服务端进入 `cloud`
- 断网时，服务端能开启 `esp32-server`
- 浏览器能打开服务端页面
- `/api/status` 返回 JSON
- `/api/control` 能控制风扇和窗帘

## 当前最重要的接口

服务端当前真正稳定可用的主接口只有两个：

- `GET /api/status`
- `POST /api/control`

不要把 MQTT 当成本轮理解项目的入口，它现在更像未来预留位。

## 调试顺序建议

### 第一阶段：只看网页

- 用手机或电脑接入 `esp32-server` 或同一局域网
- 打开服务端控制页
- 观察窗帘、风扇和传感器反馈

### 第二阶段：只看硬件稳定性

- 先单独测试风扇
- 再单独测试窗帘舵机
- 最后一起测试，关注供电、共地和干扰

### 第三阶段：再看客户端

- 确认服务端接口稳定后，再接入 `esp32_home_client`
- 如果客户端行为和服务端 README 不一致，以服务端当前实现为准

## 当前应当优先排查的问题

如果现场出现异常，优先顺序通常是：

1. 供电问题
   直流电机、舵机和 ESP32 是否共电且压降明显。
2. 接线问题
   是否共地，驱动器是否正确接入，串口是否交叉。
3. 服务端日志
   看是否有 Wi-Fi 重连、传感器错误、串口异常。
4. 协议问题
   看 `/api/status` 和 `/api/control` 是否返回符合预期。

## 你可以暂时忽略的内容

- `esp32_ir_Study/`
  它不是当前主链路。
- `esp8266_ir_control/` 的现有代码
  目前还是占位工程。
- MQTT 细节
  当前不是必读，也不是必跑。
